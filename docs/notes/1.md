建议你按“先出作品，再理解节点”的路线学，不要一开始陷进节点大全。

**学习路线**
1. **第一阶段：文生图**
   目标：稳定出 20 张图，学会改 prompt、seed、steps、CFG、分辨率。
   推荐工作流：`SDXL txt2img` 或 `Flux Schnell/Dev FP8 checkpoint`。
   ComfyUI 官方例子里，SDXL 推荐 1024x1024 或等像素量比例；Flux 也有单文件 FP8 checkpoint，可以直接放进 `models/checkpoints/` 用 `Load Checkpoint` 节点加载。([comfyanonymous.github.io](https://comfyanonymous.github.io/ComfyUI_examples/sdxl/))

2. **第二阶段：图生图 / 局部修改**
   目标：拿一张图做风格化、重绘、扩图、局部修复。
   推荐工作流：`img2img`、`inpainting`、`Flux Kontext`、`ControlNet Canny/Depth`。
   ComfyUI 官方 README 明确支持 inpainting、ControlNet、LoRA、upscale、workflow JSON 保存/加载，也支持把生成图片里的 workflow 再拖回 ComfyUI 复现。([github.com](https://github.com/comfyanonymous/ComfyUI))

3. **第三阶段：图生视频**
   目标：先做 3-5 秒短视频，不追求长片。
   推荐工作流：`Wan 2.1 Image to Video`。
   官方 Wan 示例要求视频 diffusion model 放 `models/diffusion_models/`，`clip_vision_h.safetensors` 放 `models/clip_vision/`；示例默认 33 帧 512x512，是为了更容易跑通。([comfyanonymous.github.io](https://comfyanonymous.github.io/ComfyUI_examples/wan/))

4. **第四阶段：文生视频**
   目标：从 prompt 直接生成短视频。
   推荐工作流：`Wan 2.1 Text to Video 1.3B`，先别上 14B。
   Wan 2.1 官方示例里，T2V 1.3B 模型放 `models/diffusion_models/`，还需要 text encoder 和 VAE；质量排序一般是 `fp16 > bf16 > fp8_scaled > fp8_e4m3fn`，显存不够再用 fp8。([comfyanonymous.github.io](https://comfyanonymous.github.io/ComfyUI_examples/wan/))

**我建议你先做这 3 个作品**
1. **头像/海报图**
   用 SDXL 或 Flux Schnell，产出 10 张，挑 1 张做最终图。

2. **商品/人物风格图**
   用图生图或 Flux Kontext，把参考图变成统一风格。

3. **3 秒动态短片**
   先用一张你满意的图，走 Wan 2.1 I2V，做轻微镜头运动。

**模型管理规则**
把模型按类型放，不要乱塞：

```text
ComfyUI/models/checkpoints/        SDXL、Flux FP8 单文件 checkpoint
ComfyUI/models/diffusion_models/   Flux full、Wan 视频模型
ComfyUI/models/text_encoders/      t5xxl、umt5 等文本编码器
ComfyUI/models/vae/                VAE
ComfyUI/models/clip_vision/        图生视频/参考图相关视觉编码器
ComfyUI/models/loras/              LoRA 风格包
ComfyUI/models/controlnet/         ControlNet / Canny / Depth
ComfyUI/models/upscale_models/     放大模型
```

ComfyUI 的默认模型目录就是这些分类；它也支持用 `extra_model_paths.yaml` 指向一个外部统一模型仓库，适合以后 Mac 和服务器共用模型目录。([raw.githubusercontent.com](https://raw.githubusercontent.com/Comfy-Org/ComfyUI/master/extra_model_paths.yaml.example))

**你的本项目建议**
当前阶段先保持简单：

```text
Mac 本机:
  ComfyUI/models/...

服务器:
  /data/wangqiao/ComfyUI/models/...
```

等模型多起来，再抽成：

```text
/data/models/comfy/
  checkpoints/
  diffusion_models/
  text_encoders/
  vae/
  loras/
```

然后用 `extra_model_paths.yaml` 统一挂载。

**下一步你就做这个**
1. 打开 ComfyUI。
2. 先用内置 template 或官方 example workflow。
3. 下载一个 SDXL 或 Flux FP8 checkpoint 到 `ComfyUI/models/checkpoints/`。
4. 跑通第一张图。
5. 保存 workflow JSON，命名成：

```text
workflows/001-sdxl-poster.json
workflows/002-flux-portrait.json
workflows/003-wan-i2v-short.json
```

你现在不要追求“会所有节点”，先追求“每周完成 3 个可展示作品”。
