
可以，远程服务器部署主线就是：

```bash
git clone --recurse-submodules git@github.com:amuqiao/comfy-shell.git
```

不需要你手工再单独 `git clone ComfyUI`。`ComfyUI/` 是这个壳项目的 git submodule，`--recurse-submodules` 会自动把子项目拉下来。

如果你已经普通 clone 了：

```bash
git clone git@github.com:amuqiao/comfy-shell.git
cd comfy-shell
git submodule update --init --recursive
```

服务器部署建议流程：

```bash
cd /data/wangqiao

git clone --recurse-submodules git@github.com:amuqiao/comfy-shell.git
cd comfy-shell

# 先只读体检
cp .env.example .env
./scripts/check_env.sh
```

如果服务器没有 `uv`，先装：

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

然后准备依赖：

```bash
UV_INDEX_URL=https://pypi.tuna.tsinghua.edu.cn/simple ./scripts/local.sh bootstrap
```

启动：

```bash
./scripts/local.sh start
./scripts/local.sh status
```

看日志：

```bash
./scripts/local.sh logs
```

停止 / 重启：

```bash
./scripts/local.sh stop
./scripts/local.sh restart
```

远程访问不要直接暴露公网端口。当前 server `.env` 监听的是：

```text
COMFY_HOST=127.0.0.1
COMFY_PORT=8188
```

所以在你的 Mac 上开 SSH 隧道。执行前先确认本机 `.env` 有真实 SSH 目标：

```dotenv
REMOTE_HOST=wangqiao@47.94.108.140
REMOTE_TUNNEL_LOCAL_PORT=8188
REMOTE_TUNNEL_REMOTE_HOST=127.0.0.1
REMOTE_TUNNEL_REMOTE_PORT=8188
```

```bash
./scripts/remote.sh tunnel
```

然后浏览器打开：

```text
http://127.0.0.1:8188
```

几个关键点：

- 壳仓库不包含 ComfyUI 代码全文，只记录 submodule 指针。
- 服务器 clone 时用 `--recurse-submodules`，子项目会自动拉。
- 模型不会随 git 下载，后续按 ComfyUI 页面缺失提示、Manager，或 `scripts/models.sh` 手动下载。
- `.env` 不提交；脚本默认读取当前 checkout 根目录 `.env`。
- `ComfyUI-Manager` 不是单独 clone 到 `custom_nodes`，当前方式是安装 `ComfyUI/manager_requirements.txt`，启动时用 `--enable-manager`。

如果本地已经有 checkout，也可以用本机 `.env` 中的 `REMOTE_*` 遥控同一台服务器。
执行前先确认：

```dotenv
REMOTE_HOST=wangqiao@47.94.108.140
REMOTE_DIR=/data/wangqiao/comfy-shell
REMOTE_READY_URL=http://127.0.0.1:8188
REMOTE_TUNNEL_LOCAL_PORT=8188
REMOTE_TUNNEL_REMOTE_HOST=127.0.0.1
REMOTE_TUNNEL_REMOTE_PORT=8188
```

```bash
./scripts/remote.sh sync --yes
ssh wangqiao@47.94.108.140 'cd /data/wangqiao/comfy-shell && cp .env.example .env'
./scripts/remote.sh bootstrap --yes
./scripts/remote.sh start --yes
./scripts/remote.sh status
./scripts/remote.sh tunnel
```

需要临时自定义服务器或目录时，再用 `--host` 和 `--dir` 覆盖本次调用：

```bash
./scripts/remote.sh status --host wangqiao@服务器IP --dir /data/wangqiao/comfy-shell
```
