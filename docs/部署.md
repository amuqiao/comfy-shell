
可以，远程服务器部署主线就是：

```bash
git clone --recurse-submodules git@github.com:amuqiao/comfy-shell.git
```

不需要你手工再单独 `git clone ComfyUI`。`ComfyUI/` 是这个壳项目的 git submodule，`--recurse-submodules` 会自动把子项目拉下来。

如果你已经普通 clone 了：

```bash
git clone git@github.com:amuqiao/comfy-shell.git
cd comfy-shell
git submodule update --init --recursive
```

服务器部署建议流程：

```bash
cd /data/wangqiao

git clone --recurse-submodules git@github.com:amuqiao/comfy-shell.git
cd comfy-shell

# 激活服务器 CUDA profile，会生成 .env
./scripts/env.sh use server-cuda-a10

# 先只读体检
./scripts/check_env.sh
```

如果服务器没有 `uv`，先装：

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

然后准备依赖：

```bash
UV_INDEX_URL=https://pypi.tuna.tsinghua.edu.cn/simple ./scripts/local.sh bootstrap
```

启动：

```bash
./scripts/local.sh start
./scripts/local.sh status
```

看日志：

```bash
./scripts/local.sh logs
```

停止 / 重启：

```bash
./scripts/local.sh stop
./scripts/local.sh restart
```

远程访问不要直接暴露公网端口。当前 server profile 监听的是：

```text
COMFY_HOST=127.0.0.1
COMFY_PORT=8188
```

所以在你的 Mac 上开 SSH 隧道：

```bash
./scripts/remote.sh tunnel --target server-a10
```

然后浏览器打开：

```text
http://127.0.0.1:8188
```

几个关键点：

- 壳仓库不包含 ComfyUI 代码全文，只记录 submodule 指针。
- 服务器 clone 时用 `--recurse-submodules`，子项目会自动拉。
- 模型不会随 git 下载，后续按 ComfyUI 页面缺失提示、Manager，或 `scripts/models.sh` 手动下载。
- `.env` 不提交，服务器上用 `./scripts/env.sh use server-cuda-a10` 生成。
- `ComfyUI-Manager` 不是单独 clone 到 `custom_nodes`，当前方式是安装 `ComfyUI/manager_requirements.txt`，启动时用 `--enable-manager`。

如果本地已经有 checkout，也可以用固定远程服务器入口遥控同一台服务器：

```bash
./scripts/remote.sh sync --target server-a10 --yes
./scripts/remote.sh bootstrap --target server-a10 --yes
./scripts/remote.sh start --target server-a10 --yes
./scripts/remote.sh status --target server-a10
./scripts/remote.sh tunnel --target server-a10
```

需要自定义服务器或目录时，使用可复用入口：

```bash
./scripts/remote.sh status --host wangqiao@服务器IP --dir /data/wangqiao/comfy-shell
```
